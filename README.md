## how i learn (and maybe you can too)

instead of dumping random skills, here's actually how i approach learning stuff:

### step 1: understand *why* i'm learning it
- not "because it's useful" but "because i actually want to build it"
- motivation > discipline. always.

### step 2: find the fundamentals
- skip fancy tutorials with 10 libraries
- learn the core concept first
- *then* add complexity

### step 3: build something small
- even if it's terrible
- projects > tutorials. facts.
- fail fast, learn faster

### step 4: actually use what i learned
- use it in another project
- teach someone else
- break it and fix it

### step 5: reflect
- what took longer than expected?
- what was easier than i thought?
- write it down somewhere
